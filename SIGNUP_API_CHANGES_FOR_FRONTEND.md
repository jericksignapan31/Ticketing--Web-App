# Signup API Changes - Frontend Update Guide

## 📋 Summary
The signup API has been updated to:
1. **Remove password input from frontend** - No longer needed
2. **Auto-generate temporary 6-digit password** - Backend generates and stores it
3. **Return temporary password in response** - Display to user for first login

---

    email: this.form.value.email,

    branch_id: this.form.value.branch_id,

## 🔑 Login Flow - With Password Change Tracking

Users will login with:
```json
POST /auth/login
{
  "username": "john.doe@example.com",
  "password": "482917"  // Use the temporary password
}
```

### ✅ New Login Response (Includes password_changed flag)

```json
{
  "access_token": "eyJhbGc...",
  "user": {
    "user_id": "uuid-here",
    "username": "john.doe@example.com",
    "password_changed": false,  // ✅ NEW: false = still on temp password
    "employee": {
      "employee_id": "uuid-here",
      "first_name": "John",
      "last_name": "Doe",
      "email": "john.doe@example.com",
      "role": "employee",
      "branch_id": "uuid-here"
    }
  }
}
```

### 🔐 Mandatory Password Change on First Login

The frontend **MUST** check the `password_changed` flag after login:

✅ **Frontend Logic:**
```typescript
onLoginSuccess(response: LoginResponse) {
  // ✅ Check if password needs to be changed
  if (!response.user.password_changed) {
    // First login - force password change
    alert('You must change your temporary password before continuing!');
    this.router.navigate(['/change-password']);
    return;
  }
  
  // Password already changed - allow access to dashboard
  this.router.navigate(['/dashboard']);
}
```

### 🔄 Password Change Endpoint

After first login, users must change their password:

```json
POST /auth/change-password
{
  "currentPassword": "482917",           // Temporary password
  "newPassword": "MyNewSecurePassword123!"
}
```

**Response on Success:**
```json
{
  "message": "Password changed successfully"
}
```

**Backend Update:** When user successfully changes password:
- `password_changed` flag is set to `true` in database
- Next login will have `password_changed: true` in response
- Frontend will allow normal access to dashboard

## 🎯 Complete First Login Flow

```
User Signs Up
    ↓
Backend generates 6-digit temporary password
    ↓
Frontend displays: "Your temporary password is: 482917"
    ↓
User Logs In with Email + Temporary Password
    ↓
Backend returns: { user: { password_changed: false } }
    ↓
Frontend checks: if (!password_changed)
    ↓
Frontend redirects to: /change-password page
    ↓
User enters new secure password
    ↓
Backend validates password & sets password_changed = true
    ↓
User logs in again with new password
    ↓
Backend returns: { user: { password_changed: true } }
    ↓
Frontend allows access to Dashboard ✅
```

## ⚠️ Important Notes

1. **No More Manual Employee IDs** - Frontend doesn't need to input employee_id anymore
2. **No More Password Input** - Users don't set their own password during signup
3. **Temporary Password is 6 digits** - Always a 6-digit random number (e.g., "123456")
4. **Display Password to User** - Make sure to show the temporary password clearly
5. **Advise Password Change** - Tell users to change their password after first login
6. **Email is Username** - Users login with their email address, not employee_id
7. **⚠️ Check `password_changed` Flag** - After login, frontend MUST check if `password_changed` is false
8. **⚠️ Mandatory Password Change** - If `password_changed` is false, redirect user to `/change-password` page
9. **⚠️ First Login Security** - Users cannot access dashboard until they change their temporary password
